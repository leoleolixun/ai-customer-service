## 8. 已知限制和遗留问题

- V1.0 每个应用只允许一个活动 Chat 模型，每个知识库固定一个 Embedding 模型和版本。
- V1.0 不提供 OCR、扫描 PDF、自动模型路由、完整工单、SLA 或业务写操作。
- `glm-4.5-air` 默认动态思考导致本机验收首 Token P95 未达到 5 秒目标；V1.0 不承诺供应商 SLA。
- 真实 GLM 用量已记录 Token、耗时和状态；本次模型配置未填写计价参数，因此费用估算为 `0`，不能作为
  实际账单依据。
- Redis 不保存需要长期恢复的业务事实；当前备份不包含 Redis，恢复后由数据库和幂等任务重建短期状态。

以上限制均属于冻结范围内的明确约束，不替代第 9 节列出的复验条件。
