## 7. 已知限制

- V1.0 不提供 OCR、扫描 PDF、完整工单/SLA、自动模型路由、业务写操作或商业计费。
- 每个应用只绑定一个活动 Chat 模型，每个知识库固定一个 Embedding 模型和版本。
- 免费 GLM Provider 在无间隔连续采样时曾返回一次 `429`；正式复验采用 6 秒间隔后 10/10 通过。这不构成
  供应商 SLA，生产应使用有保障的额度、监控和后续备用路由。
- Redis 不保存长期业务事实；恢复时只清空 Compose 渲染的 `APP_REDIS_URL` 所指向的 DB。
- 单机备份已验证和恢复，但还必须复制到加密的主机外故障域，才可视为耐久备份。
- 当前没有 Git Remote，因此没有云端 CI、远端镜像和 `v1.0.0` Tag。

## 8. 最终结论

- 结论：**不通过**。
- 是否允许发布 `v1.0.0` Tag：**否**。

- 发布前必须由未参与开发的人完成 30 条 RAG 复核，覆盖全部 5 类，并重新执行 `--enforce-gate`。
- 发布前必须由独立接入人从 Commit `9c34121` 的干净工作区启动计时，只依据接入文档在 30 分钟内完成
  Widget 接入，提交桌面/375px 证据和管理端会话核对记录。
- 发布前必须配置 Git Remote，推送冻结 Commit，并取得同一 Commit 全部云端 CI Job 的通过记录。

以上三项全部通过后，补充评审人、接入人、CI Run 和最终签字，再将本结论改为“通过”并发布 Tag。

### 8.1 当前人工复核交接命令

当前空白工作表仍保存在本机 `/tmp/human-review-9c34121.jsonl`，其 SHA256 已记录在第 5 节。独立评审人

```bash
uv run python scripts/human_review_server.py \
  --worksheet /tmp/human-review-9c34121.jsonl \
  --candidate 9c34121eb8744747dd6197e6d2e3b19b681ca833 \
  --expected-sha256 3fcac695b5073b0ec458ffa880bc1f0459eba14829edfd43a88a81b570771c48
```

```bash
uv run python scripts/human_review.py merge \
  --predictions /tmp/rag-predictions-9c34121.jsonl \
  --reviews /tmp/human-review-9c34121.jsonl \
  --output /tmp/rag-reviewed-predictions-9c34121.jsonl

uv run python scripts/evaluate_rag.py score \
  --predictions /tmp/rag-reviewed-predictions-9c34121.jsonl \
  --output /tmp/rag-reviewed-report-9c34121.json \
  --enforce-gate
```
